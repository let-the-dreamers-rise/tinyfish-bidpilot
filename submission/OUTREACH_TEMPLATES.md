## 4. Reply to "How does it work?" Objection

```
Great question. BidPilot works in three steps:

1. Your team builds a "vendor packet" — a reusable bundle of company data, tax forms, insurance certs, and compliance docs.

2. When you need to onboard on a new portal, BidPilot's AI browser agent (powered by TinyFish) actually opens the portal, logs in securely, navigates the multi-step forms, fills every field from your packet, uploads the documents, and saves the draft.

3. Your team gets a notification to review the saved draft before any final submission. Nothing goes live without human approval.

The key difference from standard AI tools: we don't just generate text. We actually DO the portal work — navigation, file uploads, form submissions — on the live web.

Happy to do a 10-min screen share so you can see it in action?
```

## 5. Reply to "Is it safe?" Objection

```
Totally fair concern. Security is core to how we built this:

- Portal credentials are stored in TinyFish Vault (encrypted, never on our servers)
- Default execution is "draft-save only" — the agent saves but NEVER submits without your approval
- Every action is logged in an immutable audit trail
- Database has row-level security on every table
- All data is encrypted at rest (AES-256) and in transit (TLS 1.3)

We built this for enterprise procurement teams, so compliance isn't an afterthought — it's the foundation.
```
